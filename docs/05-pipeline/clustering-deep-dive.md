## 0. TL;DR — 핵심 가설 한 줄

> **같은 이슈를 여러 언론사가 동시에 보도한다 = 공영 가치가 높다 = 주요 이슈다.**

서신은 이 가설을 **통계적 군집화(clustering)** 로 자동화합니다.
- **4개 소스**(Google News / 연합뉴스 / 서울신문 / **네이버 검색 API**)에서 기사 ~60~80건 수집
- 제목·요약 유사도로 같은 이슈끼리 묶음 (**TF-IDF + 코사인 유사도**)
- **클러스터가 클수록 = 많은 언론사가 보도 = 중요**
- 상위 3개 클러스터의 대표 기사만 LLM에 전달
- 결과: **80건 중 3건만** 비용 드는 LLM이 처리 → 10배 비용 절감 + 신호 증폭

## 1. 철학: 왜 클러스터링인가?

### 1.1 대안들과 서신의 선택

| 방식 | 비용 | 문제점 |
|---|---|---|
| ① 전체를 LLM에 "이 중 중요한 거 골라줘" | $$$ 비쌈 (80건 분석) | 경량 LLM은 판단 부정확 + 토큰 폭주 |
| ② 최신순 상위 N | 0 | "최신 = 중요" 가정 틀림 |
| ③ 포털 인기순/조회수 | 0 | RSS는 조회수 제공 X + 인기 ≠ 공영성 |
| ④ **TF-IDF 클러스터링 + 크기 기반 선별** | 0 | ⭐ 서신 선택 |

**④ 선택 이유**:
1. **비용 0** — 순수 수학/통계 (scikit-learn, CPU 몇 ms)
2. **공영 가치의 수학적 정의** — "다수 매체 교차 보도" = 의사결정 합의
3. **경량 LLM의 한계를 코드가 보완** — 서신 전체 철학과 일관

### 1.2 "코드가 할 수 있는 건 코드가"

서신의 설계 철학:
- **의미 이해**(3줄 요약, 라디오 스크립트) → LLM
- **중요도 판정, 중복 제거, 출처 다양성 확보** → 코드 (TF-IDF + 그리디)

이 역할 분리 덕에 **gpt-5-nano 같은 경량 모델로도 프로덕션 품질**이 나옵니다.

## 3. 각 단계 상세 설명

### 3.1 제목 정규화 (`normalize_title`)

#### 문제: 편집 메타데이터가 유사도를 왜곡

뉴스 제목은 순수한 내용만 있는 게 아닙니다:

| 편집 메타 유형 | 예시 |
|---|---|
| 접두사 | `[종합]`, `[속보]`, `[단독]`, `[사설]`, `(1보)`, `(종합)`, `(단독)` |
| 접미사 | `- 조선일보`, `- 연합뉴스`, `- 사이언스타임즈`, `- 뉴시스` |

TF-IDF가 이 토큰들을 "의미 있는 단어"로 간주하면 **가짜 유사도**가 생깁니다:

- `[종합] A 기사 - 연합뉴스` vs `[종합] B 기사 - 연합뉴스`
- 둘이 전혀 다른 주제인데 `[종합]` + `연합뉴스` 공통 → 유사도 **인위적으로 +0.1~0.2**
- → 같은 클러스터로 잘못 묶이거나, 유사도 순위가 왜곡

#### 해법: regex로 노이즈 제거

```python
_TITLE_PREFIX_NOISE = re.compile(r"^\s*(?:\[[^\]]{1,15}\]|\([^)]{1,15}\))\s*")
_TITLE_SUFFIX_NOISE = re.compile(
    r"\s*[-–—]\s*"
    r"[가-힣A-Za-z0-9·\s]{1,20}"
    r"(?:일보|뉴스|타임즈|경제|방송|TV|신문|미디어|데일리|NEWS|뉴시스|통신|사이언스타임즈)"
    r"\s*$"
)

def normalize_title(title):
    t = title
    for _ in range(3):                     # 스택된 접두사 처리
        stripped = _TITLE_PREFIX_NOISE.sub("", t)
        if stripped == t: break
        t = stripped
    t = _TITLE_SUFFIX_NOISE.sub("", t)
    t = re.sub(r"\s+", " ", t).strip()
    return t
```

#### 실측 예시

| Before | After |
|---|---|
| `[종합] 메모리 반도체 호황 - 조선일보` | `메모리 반도체 호황` |
| `[단독][종합] AI 속도 제어 - 사이언스타임즈` | `AI 속도 제어` |
| `(1보) 모디 정상회담 - 뉴시스` | `모디 정상회담` |

### 3.2 Collector-level 중복 제거

`MultiSourceCollector.fetch()` 에서 **2단계 dedup**:

1. **URL 기반** — 완전 동일 link 1회만 통과
2. **정규화된 제목 기반 (NEW)** — 다른 URL인데 제목이 거의 같은 경우 (예: 네이버가 연합뉴스 기사 미러링) 둘 다 들어오는 문제 해결

```python
seen_urls = set()
seen_title_keys = set()
for a in merged:
    if a.link in seen_urls: continue
    title_key = normalize_title(a.title).strip().lower()
    if title_key in seen_title_keys: continue
    seen_urls.add(a.link)
    seen_title_keys.add(title_key)

```

**효과**: TF-IDF 클러스터링 전에 이미 **같은 기사의 재배포본이 하나로 수렴** → 클러스터 크기 신호가 더 정확해짐.

### 3.3 TF-IDF 벡터화

#### 왜 TF-IDF인가?

- **Term Frequency (TF)**: 문서 내 단어 출현 빈도 — "자주 나오는 단어가 중요"
- **Inverse Document Frequency (IDF)**: 여러 문서에 나타나는 단어는 덜 중요 — "흔한 단어는 변별력 낮음"
- **TF × IDF**: 한 문서에서 자주 나오면서 전체 코퍼스에선 드문 단어 = **특정 이슈 식별자**

예: "인도"라는 단어가 "대통령 인도 순방" 기사에 4번 나오고 (TF 높음), 전체 60개 기사 중 5개에만 나온다면 (IDF 높음) → "인도"는 해당 이슈의 강한 식별자.

#### 서신의 TF-IDF 설정

```python
def _article_text_for_clustering(a):
    title = normalize_title(a.title)
    summary = _clean(a.summary or "")[:300]
    return f"{title} {title} {summary}"   # 제목 2번 → 가중

TfidfVectorizer(
    tokenizer=_tokenize_ko,      # [\w가-힣]+ 2자 이상
    lowercase=False,
    token_pattern=None,
    ngram_range=(1, 2),          # unigram + bigram
    sublinear_tf=True,           # 1 + log(tf), 빈출 억제
    min_df=1,                    # 희귀 단어 유지
)
```

| 파라미터 | 선택 이유 |
|---|---|
| `tokenizer=_tokenize_ko` | 한국어 어절 단위 (regex로 단순하지만 충분) |
| `ngram_range=(1,2)` | **Bigram까지 포함** — 조사/어미 차이에 강건 (예: "대통령 인도" 자체가 하나의 특징) |
| `sublinear_tf=True` | `1 + log(tf)` 변환. 같은 단어가 10번 나와도 `1 + 2.3 = 3.3` 정도로 완만. 빈출 filler 어휘 가중치 완화 |
| 제목 중복 `"{title} {title} {summary}"` | 제목은 핵심 정보 응축체. 요약 300자가 더 많은 토큰이 있기 때문에 그냥 합치면 요약이 더 영향력 커짐. 제목 중복으로 약 2배 가중 |

#### 출력

- 60개 문서 × ~2,000 features 의 **희소 행렬** (scipy sparse)
- 대부분 0 (각 문서는 전체 vocabulary의 극히 일부만 씀)
- 메모리 효율적

### 3.4 코사인 유사도

```python
sim = cosine_similarity(matrix)   # (n × n) 대칭 행렬
```

`sim[i,j] = (vec_i · vec_j) / (|vec_i| × |vec_j|) ∈ [0, 1]`

- 0 = 완전 다른 주제
- 1 = 완전 동일
- 보통 같은 이슈끼리는 **0.4~0.8** 범위

### 3.5 그리디 클러스터링

```python
visited = [False] * n
clusters = []
for i in range(n):
    if visited[i]: continue
    group = [i]
    visited[i] = True
    for j in range(i+1, n):
        if visited[j]: continue
        if sim[i, j] >= 0.45:      # threshold
            visited[j] = True
            group.append(j)
    clusters.append(group)
```

#### 왜 그리디?

- **k-means**: k를 미리 정해야 함 (이슈 개수가 매번 다름)
- **hierarchical**: O(n² log n), 60개라 가능하지만 복잡
- **DBSCAN**: ε(epsilon) + min_samples 튜닝 필요
- **그리디**: 단순, O(n²), 충분히 좋은 근사 — 뉴스 데이터는 매일 바뀌어서 정밀성보다 **강건성** 우선

#### 왜 threshold 0.45?

- Before (0.6): 정규화 전 기준. 같은 이슈도 표현 달라지면 놓침
- Now (0.45): 정규화 + bigram으로 **진짜 신호** 비율이 올라가서 더 관대하게 묶을 수 있음

#### Single-pass 주의점

순서 의존성: `i`를 먼저 뽑은 뒤 `j`를 검사하므로, A-B가 0.4 / B-C가 0.5 / A-C가 0.6 일 때
- A → C (0.6 ≥ 0.45) 묶음 → B는 별도 클러스터로 분리
- 이론적 불완전성이지만 **hierarchical까지 쓸 이득이 적음** (정확성보다 해석 가능성과 속도)

### 3.6 클러스터 내 대표 기사 선별 (`_select_representatives`)

```python
def _select_representatives(members, max_n=3):
    seen_sources = set()
    picked = []
    for m in members:
        if m.source in seen_sources: continue
        picked.append(m)
        seen_sources.add(m.source)
        if len(picked) >= max_n: break
    return picked
```

**규칙**:
- 같은 클러스터 안에서 **서로 다른 언론사** 우선 (출처 다양성)
- 최대 3건까지
- 최신순 (cluster 생성 직전 `members.sort(key=published desc)`)

**왜?**
- 같은 이슈를 보도한 여러 언론사 중 **관점이 다를 수 있는 최신 기사 3건**을 LLM 컨텍스트로 제공
- LLM이 **크로스 체크** 가능

### 3.7 클러스터 정렬 + Top 3 선별

```python
clusters.sort(key=lambda c: c.size, reverse=True)
```

**이게 핵심 발표 포인트**:

> 클러스터 크기 = 해당 이슈를 **독립적으로 보도한 언론사 수**
> → 여러 매체가 동의한 공영 이슈 → **중요도 signal**

`pick_top_articles(clusters, n=3)` 이 상위 3개 클러스터에서 각 1건씩 뽑음 (출처 다양성 우선).

### 3.8 안전망: Post-Select 중복 검증 (Day 5 추가)

클러스터링이 완벽하지 않을 수 있음:
- 같은 이슈가 표현 차이로 **두 클러스터로 분리**된 경우
- Top 2에 들어가면 → 최종 3건에 **같은 이슈 2개 포함**

해법: 선별 **이후** 한 번 더 검증.

```python
# 선별된 picks에 대해 pairwise 유사도 계산
for i, j in pairs(picked):
    if sim[i, j] > 0.55:
        # 비슷한 쌍의 후자를 pool의 다음 후보로 교체
        picked[j] = next_best_candidate
        continue  # 교체 후 다시 전체 pairwise 재검증
```

**하드 리밋**: 최대 4회 교체 — 병리적 입력에서 무한 루프 방지.

## 5. 설계 결정 요약 (Q&A 대비)

| 질문 | 답변 |
|---|---|
| **Q. 왜 임베딩(KoBERT) 안 썼나요?** | 비용/환경 제약. 임베딩은 모델 다운로드 수백 MB + 추론 시간 + 서버 메모리. 서신은 경량화 우선. 게다가 뉴스 제목은 TF-IDF가 의외로 잘 작동 (키워드 기반 매칭). 추후 확장 시 KoBERT/KLUE-RoBERTa 교체 설계는 가능 (`_article_text_for_clustering` 만 바꾸면 됨) |
| **Q. 왜 k-means 안 썼나요?** | k를 미리 정해야 함. 매일 뉴스 이슈 개수가 달라서 부적합. 그리디는 threshold만 정하면 자동 |
| **Q. threshold 0.45는 어떻게 정했나요?** | 정규화 + bigram + sublinear_tf 조합에서 실측 기준. 너무 낮으면(0.3) 관련 없는 이슈가 묶이고, 너무 높으면(0.6) 같은 이슈도 분리. 0.45에서 최적 (+ post-select 안전망으로 추가 방어) |
| **Q. "대통령"처럼 흔한 단어가 모든 이슈에 나오면?** | IDF가 자동으로 가중치 낮춤. 모든 문서에 나오는 단어는 IDF=0에 가까움 → 변별력 거의 없음 |
| **Q. 영어 이름 대소문자는?** | `lowercase=False` 로 원형 유지. 한국 기사에선 대문자 변화가 많지 않아 크게 영향 없음 |
| **Q. 실패 케이스는?** | ①한 단어만 있는 기사 (vocab 0) → fallback으로 개별 클러스터 ②전부 다른 주제 → 모든 클러스터 크기 1 → 최신순으로 Top 3 |

## 6. 한계 및 향후 확장

### 6.1 현재 한계

1. **통계적 유사도의 한계**
   - "금리 인하"와 "기준금리 25bp 인하"는 TF-IDF로 비슷하지만, "경기 부양"과 "금리 인하"의 인과 관계는 못 포착 (임베딩 모델이 잘함)

2. **짧은 제목 취약**
   - 제목 2단어짜리는 TF-IDF가 무력 (features 부족). summary 포함으로 완화했지만 한계 있음

3. **Greedy의 순서 의존**
   - 완벽한 최적 클러스터링 아님 (hierarchical clustering은 더 나음)

### 6.2 확장 방향

| 확장 | 효과 | 비용 |
|---|---|---|
| KoSentenceBERT 임베딩 | 의미 유사도 정확 ↑↑ | 모델 500MB, CPU 추론 100ms/기사 |
| LLM 기반 클러스터링 (OpenAI `/embeddings`) | 최고 정확 | 비용 ($0.0001/1K tokens 정도) |
| 다단계 클러스터링 (LSH + re-rank) | 대규모 확장성 | 구현 복잡 |
| 사용자 개인화 | 관심사 가중치 | UI + DB 스키마 필요 |

**현재 구조의 확장 용이성**:
- `_article_text_for_clustering()` 반환값을 임베딩 벡터로 바꾸기만 하면 됨
- 나머지 파이프라인(그리디 군집화, 선별)은 동일

## 7. 발표 스크립트 (2분 핵심 메시지)

> "서신의 핵심은 **수십 건의 뉴스 중 주요 이슈를 자동 판별하는 알고리즘**입니다.
>
> 전제는 간단합니다. **여러 언론사가 같은 이슈를 동시에 보도한다면, 그건 공영 가치가 높은 주요 뉴스다.**
>
> 이 가설을 구현하기 위해 3단계 파이프라인을 씁니다.
>
> **1단계**. 4개 소스(Google News / 연합뉴스 / 서울신문 / [예정: 네이버])에서 카테고리당 최대 80건을 수집. URL과 정규화된 제목 기반으로 중복 제거. 편집 메타데이터(`[종합]`, `- 언론사명`)는 regex로 먼저 깎아냅니다. 여기서 빠뜨리면 가짜 유사도가 생깁니다.
>
> **2단계**. 제목과 요약을 TF-IDF로 벡터화합니다. bigram까지 포함하고 sublinear TF로 빈출 단어는 완화합니다. 코사인 유사도 0.45 이상이면 같은 이슈로 묶습니다. 60건이 약 25개 클러스터로 정리됩니다.
>
> **3단계**. 클러스터를 **크기 내림차순**으로 정렬합니다. **크기가 곧 중요도**입니다. 5개 언론사가 동시에 보도한 이슈는 2개가 보도한 이슈보다 중요합니다. 상위 3개 클러스터에서 서로 다른 언론사의 대표 기사 하나씩을 뽑습니다. 마지막으로 선별 결과의 pairwise 유사도를 한 번 더 검증해, 클러스터링이 놓친 중복은 여기서 제거됩니다.
>
> 결과는 **80건 중 3건**. 이 3건만 LLM이 처리합니다. **비용 95% 절감 + 공영 가치 기반 선별**. 이것이 서신이 경량 LLM으로도 프로덕션 품질을 내는 방법입니다."
